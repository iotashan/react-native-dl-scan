# Test fixtures

Synthetic fixture images for headless parity smoke tests. Used by:

- `model-training/scripts/parity_check.sh` — runs the FP32 .pt baseline,
  Core ML int8, and TFLite int8 against the same image and asserts that
  outputs are within tolerance.
- iOS XCTest target (Phase 5+) and Android instrumentation tests once an
  example app is in place.

## Provenance

All images here are **synthetic** identity documents generated by the
[IDNet](https://huggingface.co/datasets/cactuslab/IDNet-2025) project
(cactuslab/IDNet-2025, CC-BY-4.0). No real personally identifiable
information is included; the names, photos, addresses, and document numbers
are entirely procedurally generated.

Specifically: the images checked in here come from IDNet's **held-out test
split** (the same split the model was evaluated against; never seen during
training). The exact filenames + ground-truth labels are reproducible from
the IDNet 2025 dataset using `model-training/idnet/extract_subsets.py` with
seed 42 plus `model-training/idnet/prepare_yolo_fields.py` with seed 42.

## Files

| File | Description |
|---|---|
| `sample_us_california_dl.jpg` | One California driver's license, 1024×768 JPEG. Source: `idnet-data/yolo_fields/images/test/us_california_dl__generated.photos_0210845.jpg` |
| `sample_us_california_dl.txt` | YOLO-format ground-truth bboxes for the above image. Each line: `<class_id> <cx> <cy> <w> <h>` normalized to [0,1]. Class index ordering matches `cpp/yolo/field_classes.cpp`. |

## Adding new fixtures

Pick from the held-out test split (NOT the train or val splits). Keep
total fixture size under ~2 MB to avoid bloating the npm package. Document
the source filename here for provenance auditing.
